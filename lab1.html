URL

Una URL, tambien conocida como direccion web, es una referencia a una pagina en una red de computadoras. Sus siglas significan Uniform Resource Identifier, o identificador de recursos uniforme.


MIME

MIME significa Multipurpose Internet Mail Extensions y este es un estandar que extiende el formato del correo electronico para que soporte elementos como caracteres no incluidos en el codigo ASCII o adjuntos de tipo distinto a texto como lo son videos o imagenes.


Metodos

En programacion, un metodo es un procedimiento que se asocia con un objeto. Los objetos programados son capaces de realizar estos procesos o metodos.


Codigos de Errores: HTTP

HTTP es el protocolo de transferencia de hipertexto, es el protocolo de aplicacion definido como estandar que permite que la web funcione. Siempre que hay una peticion HTTP por parte del cliente (Navegador web) el servidor devuelve un codigo de estado. Cuando existe alguna falla de algun tipo en aspectos como por ejemplo, las peticiones del cliente, el servidor regresa errores con una clave que lo identifica, por ejemplo: error 404. (Ferrer, V.)

  2. Cual es el proposito de los metodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE

    3. Los metodos mencionados en HTTP sirven para realizar cierta accion o proceso sobre un elemento. Usualmente se les llaman los verbos de HTTP.

  3. Que metodo HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contrasena en algun sitio? Â¿Por que?

    4. Para subir un formulario se utiliza el metodo "post".

  4. Que metodo HTTP se utiliza cuando a traves de un navegador web se accede a una pagina a traves de un URL?

    5. Cuando se accede a una pagina web a traves de un navegador se utiliza el metodo CONNECT.

  5. Un servidor web devuelve una respuesta HTTP con codigo 200. Que significa esto? Ocurrio algun error?

    6. Un servidor devuelve codigo 200 cuando los requests han sido exitosos.

  6. Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intento acceder al sitio y se encontre con un error 404? Por que?

    7. No le corresponde al desarrollador corregir el error ya que el error es por parte del servidor y eso le corresponde corregir al proveedor del servicio.

  7. Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intento acceder al sitio y se encontro con un error 500? Por que?

    8. No, es error por parte del servidor, por lo que la solucion no le corresponde al desarrollador de encontrar.

  8. Que significa que un atributo HTML5 este depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 esten desaprobados.

  9. Cuales son las diferencias principales entre HTML 4 y HTML5?

    10. Las diferencias entre HTML4 y 5 son principalmente de sintaxis, al igual que su compatibilidad con los distintos tipos de buscadores y tipos de dispositivos y sistemas operativos.

  10. Que componentes de estructura y estilo tiene una tabla?

  11. Cuales son los principales controles de una forma HTML5?
